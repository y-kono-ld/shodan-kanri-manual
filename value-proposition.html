01なぜ、商談管理の「仕組み化」が必要か

お取引先様との商談は、商品部・バイヤーにとって最も重要な業務のひとつです。しかし、日々の商談対応に追われる中で、次のような課題が積み重なっていないでしょうか。

商談内容の属人化

交渉の経緯や決定事項が担当者個人の記憶やメモに依存し、本人以外は状況を把握しづらい。

記録の散在

やり取りがメール・口頭・紙資料に分散し、あとから経緯を追うのに時間がかかる。

上長・役員による把握の難しさ

商談がどこまで進み、どのような条件で合意したのかを、都度報告に頼らざるを得ない。

法改正への対応

2026年1月の下請法改正により、価格協議の経緯を記録として残すことの重要性が増している。

02このシステムがもたらす3つの効果

EFFECT 01

商談の見える化

申請から記録確認までの全履歴をTeams・SharePoint上に一元記録。属人化していた商談の状況を、上長・役員がいつでも確認できます。

EFFECT 02

コンプライアンス強化

上長・役員による確認と、確定後の自動ロックにより、改ざんの余地がない証跡を自動的に積み上げます。

EFFECT 03

ナレッジの資産化

部門をまたいだ商談履歴が組織の資産として蓄積され、担当者の異動や新任者の育成にも活用できます。

03改正下請法(取適法)への対応

2026年1月、下請法は「中小受託取引適正化法(取適法)」へと改正されます。小売業にとっても無関係ではない、重要な制度変更です。

制度変更の主なポイント(公開情報をもとに要約)

対象基準の拡大

従来の資本金基準に加え、従業員数基準(300人超など)が新設され、対象となる企業の範囲が広がります。

価格協議への対応義務

取引先からの価格協議の求めに応じなかったり、必要な説明を行わなかったりする行為が禁止されます。

執行体制の強化

公正取引委員会に加え、事業所管省庁による「面的執行」が強化され、調査対象が広がる見込みです。

本システムによる対応
  • 商談ごとの申請・承認・記録が、日時付きでTeams・SharePoint上に自動保存されます。
  • 記録確認が完了した情報は、上長・役員の確認を経て編集不可の状態に自動的に移行し、改ざんできない証跡として機能します。
  • 記録は5年間保存する方針で運用しており、監査や照会に耐えるアーカイブ体制を整備しています。

※制度概要は政府広報オンライン・公正取引委員会等の公開情報をもとに要約したものです。詳細は一次情報をご確認ください。

04仕組みの全体像

商談は「申請」と「記録」の2段階で管理し、必要に応じて取引先による確認プロセスも組み合わせます。

第1部・商談申請
申請登録 → 上長承認 → 許可
第2部・商談記録(商談実施後)
記録入力 → 上長確認 → 役員確認 → 確認完了
オプション・先方確認プロセス
記録確定 → 取引先へ自動送付 → 先方が内容を確認・回答 → 双方合意を記録

承認・確認の依頼と回答はすべてTeams上で行われ、判定結果は申請者に自動通知されます。取引先からの回答が一定期間ない場合も、督促と社内確認への切り替えが自動で行われる設計です。

05主な機能

実際の画面に沿って、日常の使い勝手をご紹介します。

申請

迷わず入力できる専用フォーム

商談の目的・取引先・想定条件など、必要な項目を整理した専用フォーム。第1部(申請時)・第2部(商談後の記録)に分かれており、入力すべき項目が一目で分かります。

承認

Teams通知から、その場で承認

申請内容は上長へTeamsの承認カードとして届きます。商談の詳細やSharePointへのリンクもカード内に表示されるため、画面を切り替えずにその場で判断できます。

通知

判定結果を自動で通知

承認・否認の判定結果は、申請者へ自動的にチャット通知されます。確認漏れや「聞いていない」という行き違いを防ぎます。

記録確認

商談後の記録も、同じ流れで確認

商談実施後の記録に対しても、上長・役員による確認ワークフローが自動で走ります。確認が完了すると記録は編集不可となり、正式な証跡として確定します。

可視化

状況がひと目でわかる一覧

申請・承認・記録確認の状況は色分けされた一覧に反映され、部門内の商談状況をいつでも俯瞰できます。検索・絞り込みにも対応しています。

06全社展開の実績

2026年7月時点で、以下の体制が全部門で稼働しています。

全12部門への展開完了

00テストを含む全部門で、申請・承認・記録確認のワークフローが稼働中。

先方確認プロセスも全部門対応

取引先が記録内容をWeb上で直接確認・回答できる仕組みを、全部門に展開済み。

督促・例外ルートまで自動化

取引先からの回答が一定期間ないケースも、自動督促と社内確認フローで滞留を防止。

障害時は管理者へ即時通知

フローが正常に動作しなかった場合、管理者へ自動的にアラートが届く体制を整備。

07内製開発という選択

既製の商談管理パッケージを導入する方法もありますが、本システムはMicrosoft 365の標準機能(Teams・SharePoint・Power Automate)を用いて自社で構築しています。

市販パッケージ導入本システム(内製)
自社の承認体制への適合製品仕様に合わせる場合がある部門構成に合わせて個別設計
機能追加・改善のサイクルベンダーの開発計画に依存現場の要望を都度反映
利用するツール新規ツールの導入・習熟が必要使い慣れたTeams上で完結

実際に、稼働開始後も一覧表示の改善、督促リマインダーの追加、先方回答がない場合の例外ルート整備など、現場の声をもとにした機能拡張を継続しています。
